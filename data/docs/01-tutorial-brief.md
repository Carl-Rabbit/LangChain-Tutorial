# Tutorial 简介

这个 tutorial 的目标，是在 30 分钟内做出一个最小可运行的多数据源 LLM 工作流。

我们会接两类本地数据：

- SQLite：保存日程时间、地点、DDL 这类结构化信息
- AlayaLite：保存说明文档、实验要求、FAQ 这类非结构化文本

这里有一个重要约定：为了更容易观察流程，所有示例问题都尽量使用明确日期，例如 `2026-04-18 14:00`，而不是“明天下午”。

完成后，至少要能回答三类问题：

1. 只查结构化信息的问题，例如“2026-04-18 下午的实验在哪个教室”
2. 只查文本说明的问题，例如“如果 pip 安装失败，该怎么排查”
3. 同时查两边的问题，例如“参加 2026-04-18 下午的实验，我要准备什么”

如果第三类问题答不出来，通常意味着你的工作流还没有把关系库和向量库真正串起来。
